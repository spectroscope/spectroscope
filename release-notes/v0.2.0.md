# spectroscope 0.2.0

**Released 2026-07-22.** Getting started is honest now, and the desktop build is
**signed and notarized** — double-click, no warning.

## On Maven Central

```

dev.spectroscope:spectro-orchestrator:0.2.0
```

## Providers & onboarding

The old picker pretended: it showed a model list even when a backend was
unreachable and prefilled a model that didn't exist. That's gone.

- **More backends.** Added **LM Studio** and **OpenRouter** (OpenAI-compatible
  presets) and **Gemini** as a switchable chat provider. `openai` now talks to
  `api.openai.com` — the old silent swap to a local port is removed.
- **Honest status.** `/api/config` reports each provider as `ready`,
  `needs-key`, or `local`. The picker shows what's true: a real `/api/models`
  list when it can, an honest "add a key" when it can't, labelled free text when
  a local backend is down — never a fake list.
- **First run.** A one-time sheet in the web app (pick a backend before anything
  looks broken) and a matching first-run hint on the CLI.
- **Set a key from the UI.** A masked field writes the key to `~/.spectro/.env`
  at `0600` — never to `settings.json`, never inside the app bundle (so it can't
  break a code signature). The write endpoint is origin-gated; a new chat picks
  the key up with no restart.
- **The chooser lives in Settings.** The full provider + real-model chooser is
  in Settings, with all six backends and inline key entry. The header picker
  stays a quick switcher and, when a backend needs a key, points to Settings —
  the key write lives in exactly one place.
- **One key, chat *and* image.** A `GEMINI_API_KEY` (or `OPENAI_API_KEY`) set in
  the UI feeds both the chat provider and image generation — no separate env.
- **Image generation in Settings.** Pick the image backend *and* the image model
  from a per-backend dropdown (was provider-only).
- **Local backends behave.** Switching to a reachable ollama snaps to a real
  installed model instead of a stale one; an unreachable LM Studio shows a
  working local default with a "not reachable" note, not a leftover cloud model.

## Desktop

- **Signed + notarized.** `scripts/build-desktop-runkit.sh` signs the app **and**
  the bundled JRE with a Developer ID + hardened runtime, then notarizes and
  staples the `.dmg`. It auto-detects the identity (and refuses the Valtech
  corporate cert), falling back to ad-hoc signing on machines without a cert.
  See [docs/DESKTOP-SIGNING.md](../docs/DESKTOP-SIGNING.md).

## Operating it

- **`./spectro-serve`** — a house-style launcher for the server: `start` / `stop`
  / `restart` / `status` / `logs` / `doctor` / `open`, an `--env-file` flag, and
  `set-key` (type a key hidden, saved to `.env` at `0600`).

## Docs

- User-guide sweep: new chapters for fleet control, the desktop run kit, provider
  onboarding, a "Releases & source" section, a "Saving a key, safely" chapter,
  and a freshness pass across the book. On-screen editions default to 125% zoom.

## Notes

the header picker's live-switch still accepts only the original three providers
(a switch to lmstudio/openrouter/gemini from the picker is refused; Settings
handles all six).
