## Providers & onboarding

The old picker pretended. It showed a model list even when a backend was
unreachable and prefilled a model that didn't exist. That's gone.

- **More backends.** Added **LM Studio** and **OpenRouter** (OpenAI-compatible
  presets), and **Gemini** as a switchable chat provider. `openai` now talks to
  `api.openai.com` ‚Äî the old silent swap to a local LM Studio port is removed.
- **Honest status.** `/api/config` reports each provider as `ready`,
  `needs-key`, or `local`. The picker shows what's true: a real `/api/models`
  list when it can, an honest "add a key" when it can't, free text (labelled)
  when a local backend is down ‚Äî never a fake list.
- **First run.** A one-time sheet in the web app (pick a backend before anything
  looks broken) and a matching first-run hint on the CLI instead of a terse
  missing-key error.
- **Set a key from the UI.** A masked field writes the key to `~/.spectro/.env`
  at `0600` ‚Äî never to `settings.json`, never inside the app bundle, so it can't
  break a code signature. The write endpoint is origin-gated (local requests
  only). A new chat picks the key up with no restart.
- **The chooser lives in Settings.** The full provider + real-model chooser is
  now in Settings too, with all six backends and inline key entry. The header
  picker stays a quick switcher and, when a backend needs a key, points to
  Settings ‚Äî the key write lives in exactly one place.
- **One key, chat *and* image.** A `GEMINI_API_KEY` (or `OPENAI_API_KEY`) set in
  the UI now feeds both the chat provider and image generation ‚Äî no separate env.
- **Image generation in Settings.** Pick the image backend *and* the image model
  from a per-backend dropdown (was provider-only).
- **Fixes.** Unreachable local backends no longer borrow the OpenAI model list.
  Switching to a local backend snaps to a real installed model instead of pinning
  a stale cross-provider one (e.g. opus left over from Anthropic).

## Desktop

- **Signed + notarized.** `scripts/build-desktop-runkit.sh` now signs the app
  **and** the bundled JRE with a Developer ID + hardened runtime, then notarizes
  and staples the `.dmg` ‚Äî double-click, no warning. It auto-detects the identity
  (and refuses the Valtech corporate cert), falling back to ad-hoc signing on
  machines without a cert. See [docs/DESKTOP-SIGNING.md](../docs/DESKTOP-SIGNING.md).
- App icon + a valid ad-hoc signature already landed as a 0.1.0 fast-follow (the
  first `.dmg` read as "damaged"); 0.2.0 makes it a proper signed build.

## Operating it

- **`./spectro-serve`** ‚Äî a house-style launcher for the server: `start` / `stop`
  / `restart` / `status` / `logs` / `doctor` / `open`, an `--env-file` flag, and
  `set-key` (type a key hidden, saved to `.env` at `0600`).
