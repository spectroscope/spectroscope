# spectroscope 0.1.0

**Released 2026-07-22.** The first public cut — the agent orchestrator you can watch.

## What it is

spectroscope runs LLM agents and lets you *see* every step: each event fans out
like a spectral line, each run is a spectrum, and a whole fleet lands on one
screen. Two capabilities, one runtime:

- **Harness** — a single agent: a provider, a model, tools, a permission gate.
- **Orchestrator** — a fleet: many agents as processes or in code, merged into
  one event stream.

## On Maven Central

```
dev.spectroscope:spectro-core:0.1.0
dev.spectroscope:spectro-orchestrator:0.1.0
```

The five-line facade is frozen at this version:

```java
var agent = Spectro.agent()
    .model(Anthropic.opus())
    .tools(StandardTools.all())
    .build();
agent.run("…");
```

| `spectroscope-0.1.0-arm64.dmg` | the **desktop run kit** — bundles the server + a Java runtime, so no system Java is needed |

The desktop `.dmg` is macOS arm64 and **ad-hoc signed**: it opens, but the first
launch shows the "unidentified developer" prompt (right-click → Open once). A
zero-warning, notarized build lands in 0.2.0.

## In the cockpit

- **trace** — the full turn, tool calls, and the reasoning lens.
- **spectrum** — the fleet as spectral lines, one per agent.
- **gate** — the one write path: every allow/ask/deny decision is an event.
- **replay** — scrub any recorded run from its `session.jsonl`.

## Runtime

Runs on Spring, almost nothing else. Local-first: sessions are plain
`session.jsonl` files you can read, diff, and replay without a server.
